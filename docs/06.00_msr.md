## Keycloak-Side Configuration

### Step 1: Create a New Realm
1. Log in to the Keycloak Admin Console: `htttp://ansible.demo.local:8180/auth/admin`,
username: admin
password: Password123!
2. Click on "Manage realms"

3. Click on "Create realm"
4. Enter a **Name** (e.g., `ppdm-realm`) → click **Create**
<img width="833" height="584" alt="image" src="https://github.com/user-attachments/assets/96f191ac-b98c-4402-b212-71a34a11220f" />

---

### Step 2: Configure the Realm Token Settings
1. In your new realm, go to **Realm Settings** → **Tokens** tab
2. Set **Default Signature Algorithm** to `RS256`
3. Click **Save**

---

### Step 3: Create the OIDC Client for PPDM
1. Navigate to **Clients** → click **Create**
2. Fill in:
   - **Client ID**: e.g., `ppdm-client`
   - **Client Protocol**: `openid-connect`
   - **Root URL**: `https://<ppdm-hostname>`
3. Click **Save**

---

### Step 4: Configure the Client Settings
On the client's **Settings** tab:

| Field | Value |
|---|---|
| Access Type | `confidential` |
| Standard Flow Enabled | ON |
| Implicit Flow Enabled | OFF |
| Service Accounts Enabled | ON |
| Valid Redirect URIs | `https://<ppdm-hostname>/sso/callback` |
| Web Origins | `https://<ppdm-hostname>` |

Click **Save**.

---

### Step 5: Retrieve the Client Secret
1. Go to the client's **Credentials** tab
2. Copy the **Secret** value — you will need this when registering OIDC on PPDM

---

### Step 6: Create Groups in Keycloak
1. Navigate to **Groups** → click **New**
2. Create groups that map to PPDM roles, for example:
   - `ppdm-admin`
   - `ppdm-user`
   - `ppdm-readonly`
3. Click **Save** for each group

---

### Step 7: Create Users and Assign to Groups
1. Go to **Users** → click **Add user**
2. Fill in:
   - **Username** (this becomes `preferred_username` in the token)
   - Email, First Name, Last Name
   - Set **Email Verified** → ON
3. Click **Save**
4. On the **Credentials** tab → set a password → toggle **Temporary** to OFF
5. On the **Groups** tab → join the user to the appropriate group (e.g., `ppdm-admin`)

---

### Step 8: Add Protocol Mappers to Emit Required Claims
On the client, go to the **Mappers** tab → click **Create** for each mapper below:

**Mapper 1 — groups claim**

| Field | Value |
|---|---|
| Name | `groups` |
| Mapper Type | `Group Membership` |
| Token Claim Name | `groups` |
| Full group path | OFF |
| Add to ID token | ON |
| Add to access token | ON |
| Add to userinfo | ON |

**Mapper 2 — preferred_username claim**

| Field | Value |
|---|---|
| Name | `preferred_username` |
| Mapper Type | `User Property` |
| Property | `username` |
| Token Claim Name | `preferred_username` |
| Claim JSON Type | `String` |
| Add to ID token | ON |
| Add to access token | ON |

Click **Save** for each mapper.

---

### Step 9: Verify the Well-Known OIDC Endpoint
Confirm Keycloak is publishing metadata correctly by browsing to:

```
https://<keycloak-host>/auth/realms/<realm-name>/.well-known/openid-configuration
```

This URL is what you will supply as the **Auto Config URI** when registering the OIDC provider in PPDM.

## PPDM-Side Configuration

### Step 10: Register Keycloak as an SSO Identity Provider in the PPDM UI
1. Log in to the **PPDM UI** as an Administrator or Security Administrator
2. Navigate to **Administration** → **Access Control**
3. Click the **SSO** (or **Identity Providers**) tab
4. Click **Add** to create a new SSO provider
5. Fill in the required fields:

   | Field | Value |
   |---|---|
   | Provider Type | `OIDC` / `Keycloak` |
   | Auto Config URI | `https://<keycloak-host>/auth/realms/<realm-name>/.well-known/openid-configuration` |
   | Client ID | `ppdm-client` |
   | Client Secret | `<secret from Step 5>` |

6. Click **Save** / **Verify** to test the connection and register the provider

---

### Step 11: Map Keycloak Groups to PPDM Roles in the PPDM UI
1. In the PPDM UI, go to **Administration** → **Access Control**
2. Click the **Users/Groups** tab (or **SSO Group Mapping** section)
3. Click **Add** to create a new group-to-role mapping
4. Map each Keycloak group to the corresponding PPDM role:

   | Keycloak Group | PPDM Role |
   |---|---|
   | `ppdm-admin` | Administrator |
   | `ppdm-user` | User |
   | `ppdm-readonly` | Restore Operator |

5. Click **Save** for each mapping
