# PPDM Integration with keycloak

prerequisite:
start ppdm-1
## keycloak startup on ansible host
We first refresh 0499 from github:

```bash
git -C ~/workspace/0499 pull
```

Now we start keycloak using podman

```bash
cd ~/workspace/0499/lab6/keycloak
podman-compose up -d
```

## Keycloak-Side Configuration

### Step 1: Create a New Realm
1. Log in to the Keycloak Admin Console: `htttp://ansible.demo.local:8180/auth/admin`,
username: admin
password: Password123!
2. Click on "Manage realms"
<img width="1253" height="364" alt="image" src="https://github.com/user-attachments/assets/b099c781-8842-4878-9c02-399805e402bd" />
3. Click on "Create realm"
4. Enter a **Name** (e.g., `ppdm-realm`) → click **Create**
<img width="833" height="584" alt="image" src="https://github.com/user-attachments/assets/96f191ac-b98c-4402-b212-71a34a11220f" />

---

### Step 2: Configure the Realm Token Settings
1. In your new realm, go to **Realm Settings** → **Tokens** tab
2. Check **Default Signature Algorithm** to `RS256`
<img width="1734" height="807" alt="image" src="https://github.com/user-attachments/assets/f3dc502e-0482-4ddc-83bf-ef033b911bdf" />

3. Click **Save** if required

---

### Step 3: Create the OIDC Client for PPDM
1. Navigate to **Clients** → click **Create**
<img width="1745" height="771" alt="image" src="https://github.com/user-attachments/assets/5ffc6bab-5385-44d8-83cc-7cac860b705d" />

2. On General Settings, Fill in:
   - **Client ID**: e.g., `ppdm-client`
   - **Client Type**: `openid-connect`
   - **Name**: `Client for PPDM`
3. Click **Next**
4. on Capability Config, select:


| Field | Value |
|---|---|
| Client authentication | ON |
| Standard Flow Enabled | ON |
| Standard Token Exchange | ON |
| Service Accounts Enabled | ON |

<img width="1124" height="582" alt="image" src="https://github.com/user-attachments/assets/c3aa0b77-f30a-4762-b2d0-14325f4bd3be" />
5. Click **Next**
6. on Login Settings

| Field | Value |
|---|---|
| Root URL | `hhttps://ppdm-1.demo.local` |
| Valid Redirect URIs | `will be filled when ppdm IDP is configured` |
| Web Origins | `https://ppdm-1.demo.local` |

<img width="1464" height="669" alt="image" src="https://github.com/user-attachments/assets/b5936359-9813-47b7-b435-26566aee86d4" />

Click **Save**.

---

### Step 5: Retrieve the Client Secret
1. Go to the client's **Credentials** tab
2. Copy the **Secret** value — you will need this when registering OIDC on PPDM
<img width="1142" height="514" alt="image" src="https://github.com/user-attachments/assets/22e94f93-0ee6-4c30-a3e9-0e1a87732150" />

---

### Step 6: Create Groups in Keycloak
1. Navigate to **Groups** → click **New**
2. Create groups that map to PPDM roles, for example:
   - `ppdm-admin`
   - `ppdm-user`
   - `ppdm-restore`
3. Click **Save** for each group
<img width="1753" height="478" alt="image" src="https://github.com/user-attachments/assets/97954b40-ca6d-4000-81fd-f26d0545f385" />

---

### Step 7: Create Users and Assign to Groups
1. Go to **Users** → click **Create new User**
2. Fill in:
   - **Username** (this becomes `preferred_username` in the token): ppdm-admin

   - Email: admin@demo.local
   - First ppdm
   - Last admin
   - Set **Email Verified** → ON
3. on Join Groups, select ppdm-admin
<img width="1288" height="718" alt="image" src="https://github.com/user-attachments/assets/55d34f0e-f85c-44aa-bc6e-74b7713dfd1c" />

4. Click **Create** 
5. On the **Credentials** tab → set a password → toggle **Temporary** to OFF
<img width="558" height="318" alt="image" src="https://github.com/user-attachments/assets/db3c0609-b1c9-45da-88d4-f50958c4ca2a" />


---

### Step 8: Add Protocol Mappers to Emit Required Claims
On the client, go to the **RClient scopes** tab → click **ppdm-client-dedicted**
<img width="1477" height="480" alt="image" src="https://github.com/user-attachments/assets/f6376b4b-8044-4c14-96f8-987f35c0efdd" />

Configure the following new Mappers

Mapper Type: `Group Membership` 
| Field | Value |
|---|---|
| Name | `groups` |
| Token Claim Name | `groups` |
| Full group path | OFF |
| Add to ID token | ON |
| Add to access token | ON |
| Add to userinfo | ON |
<img width="1191" height="818" alt="image" src="https://github.com/user-attachments/assets/fea31cf5-cd3e-4648-aa3e-c3e285c4a0cd" />
Click **Save**

Mapper Type: `User Property`
| Field | Value |
|---|---|
| Name | `preferred_username` |
| Property | `username` |
| Token Claim Name | `preferred_username` |
| Claim JSON Type | `String` |
| Add to ID token | ON |
| Add to access token | ON |
<img width="1150" height="878" alt="image" src="https://github.com/user-attachments/assets/096e3117-68b8-4765-927d-b1712f1c5960" />

Click **Save** 

---

### Step 9: Verify the Well-Known OIDC Endpoint
Confirm Keycloak is publishing metadata correctly by browsing to:

http://ansible.demo.local:8180/realms/ppdm-realm/.well-known/openid-configuration


This URL is what you will supply as the **Auto Config URI** when registering the OIDC provider in PPDM.

---

## PPDM-Side Configuration

### Step 10: Register Keycloak as an SSO Identity Provider in the PPDM UI
1. Log in to the **PPDM UI** as admin
2. Navigate to **Administration** → **Access Control**
3. Click the **Single Sign On**  tab
4. Click **Add** to create a new SSO provider
5. Fill in the required fields:

   | Field | Value |
   |---|---|
   | Name | `Keycloak` |
   | Identity Provider | `Okta` |
<img width="742" height="510" alt="image" src="https://github.com/user-attachments/assets/777a4c32-7bcd-4499-a6bc-f40397363031" />

6. click **next**  
7. Check I have configured ....
<img width="752" height="386" alt="image" src="https://github.com/user-attachments/assets/ede52d32-38e4-429d-86f5-4065dd7acd3f" />
 8. click **next**  

   | Field | Value |
   |---|---|
   | Client ID | `ppdm-client` |
   | Client Secret | `<secret from Step 5>` |
   | Meta Discovery URL | `http://ansible.demo.local:8180/realms/ppdm-realm/.well-known/openid-configuration` |
   <img width="1029" height="709" alt="image" src="https://github.com/user-attachments/assets/d989f1b0-48e6-4216-8939-ef942c3e70c1" />

4. Select AD/LDAP/SSO Groups /Keycloak /ppdm-admin
